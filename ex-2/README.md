# English

The starting point of the excersise can be found in the directory `begin`.

The goal of the excercise is to make use of `built-in Array methods`:
    - Currently the student instances are first assigned to a variable and then the variable is included in a manually created array that is assigned to `students`. Instead of this approach, which would require us to create a `student5` variable to add a new student, we can use the `push` method to add a new student to the array.
    - Currently a for loop is used to iterate over the array of students. Instead of this approach, we can use the `forEach` method to loop over an array instead. This is generally considered easier to read and understand.

The `solution` folder contains the solution to the excersise. It should solely be used as a directory in which a viable solution to the exersise can be found to compoare to your own solution, it should not be used as a directory in which you look for the solution to the exersise.

The end result of the excersise, will have the exact same output as the begin situation.
From the root of the repository, executing the following commands will have the same output in the console:

```sh
node ex-2/begin/index.js;
```

```sh
node ex-2/own-solution/index.js;
```

# Nederlands

Het startpunt van de oefening kan worden gevonden in de map `begin`.

Het doel van de oefening is om gebruik te maken van `ingebouwde Array-methodes`:
    - Momenteel worden de studentinstanties eerst toegewezen aan een variabele en vervolgens wordt de variabele opgenomen in een handmatig aangemaakte array die wordt toegewezen aan `students`. In plaats van deze aanpak, waarbij we een `student5` variabele zouden moeten aanmaken om een nieuwe student toe te voegen, kunnen we de `push` methode gebruiken om een nieuwe student aan de array toe te voegen.
    - Op dit moment wordt er een for-lus gebruikt om de array van `students` te doorlopen. In plaats van deze aanpak, kunnen we de `forEach` methode gebruiken om over een array te lopen. Dit wordt over het algemeen beschouwd als eenvoudiger te lezen en te begrijpen.

De `solution` map bevat de oplossing van de oefening. Deze map dient alleen gebruikt te worden als map waarin een mogelijke oplossing staat om te vergelijken met de eigen oplossing, het is niet de bedoeling om deze map te bekijken om de oefening zelf op te lossen.

Het eindresultaat van de oefening, zal exact dezelfde output hebben als de beginsituatie.
Vanuit de root van de repository, zal het uitvoeren van de volgende commando's dezelfde output in de console geven:

```sh
node ex-2/begin/index.js;
```

```sh
node ex-2/own-solution/index.js;
```
